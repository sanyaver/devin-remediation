Playbook
playbook_id → every session

7-step standard procedure enforced at the session level — not the prompt. Cannot open a PR if tests are failing. All future sessions inherit it without touching pipeline code.

Knowledge
knowledge_ids → retrieved semantically

Persistent codebase context: where tests live, which type-checking pattern this repo uses, formatting rules. Each note makes every future session smarter — the system compounds.

Schedule
cron: "0 6 * * 1"

Registered on first boot via API. Every Monday 06:00 UTC, Devin scans for new findings and opens issues automatically. Proactive, not just reactive.

Metrics API
/metrics/sessions + /metrics/prs

Dashboard analytics strip calls Devin's org-level metrics API on every load. Sessions created, PRs opened, PRs merged — live, not cached. Session insights auto-categorise completed work.

Devin isn't calling a function to edit a file — it's operating the computer like a human would. Reading issues, navigating the repo, running tests, reading the output, iterating.

Not just bug fixes

Any describable task routes through this pipeline. New API endpoints from a spec. Database migrations. New features from a ticket. The harness doesn't change — only what's in the issue.

True parallelism

Human engineers context-switch — one task at a time, attention tax on every switch. Cursor makes one engineer faster. This dispatches N engineers simultaneously — four findings, four sessions, four PRs in parallel. No routing, no assignment, no sprint planning.

Real customer engagement

What's next

01
Auto-merge on green CI for low-risk categories
Dependency upgrades and test additions are low blast radius. If CI is green and the change stays within the playbook's scope, merge automatically. Engineers only review feature-level changes. Cuts the PR review queue by ~40% in a mature pipeline.
02
Connect real signal sources — close the loop end-to-end
Snyk fires CVE alerts → webhook → Devin fixes it → PR merged before the daily standup. Coverage reporter drops below threshold → Devin writes the missing tests. APM flags a slow endpoint → Devin profiles and patches it. The pipeline is already generic. Swap the sender, not the receiver.
03
Knowledge that compounds over time
After each successful remediation, a new knowledge note: which file, which pattern, what the test for it looks like. Over time the knowledge base becomes a detailed map of the codebase's conventions. Devin's accuracy improves without touching pipeline code. New engineers onboard faster. The system gets smarter on its own.
